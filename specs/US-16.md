# US-16 · Массовое обновление и удаление контактов

**Предусловие**: в списке есть контакты (см. [US-11](./US-11.md)).

## Цель
Выделить несколько контактов и применить к ним одно действие: массовое изменение поля (например, стадии) или удаление.

## Точка входа
Сайдбар → **«Лиды»** → **«Список»** → **«Выбрать»** → выделить чекбоксами → bulk-action bar.

## Действия
- **«Удалить»** — подтверждение → `contact.deleteContacts`.
- **«Обновить поле»** / bulk-edit (напр. сменить стадию у всех) → `contact.bulkUpdate`.

## Что вызывается
- `contact.bulkUpdate` — input `{ workspaceId, contactIds, patch }` (форма `patch` — см. contracts, N=1 в capture).
- `contact.deleteContacts` — input `{ workspaceId, contactIds }`.

Сигнатуры — см. [`api-contracts.md`](./api-contracts.md).

## Success-flow
- Список инвалидируется (Firestore-подписка сама обновит).
- Bulk-mode закрывается.
- Toast об успехе (текст в capture не зафиксирован — [OQ-1](#открытые-вопросы)).

## Error-flow
`toast.error(err.message)`, выделение остаётся.

## Открытые вопросы
- **OQ-1**: точный текст success-toast для bulk-update / bulk-delete.
- **OQ-2**: ограничение на максимальный batch size — есть ли лимит?
- **OQ-3**: есть ли подтверждение на удаление (confirm-модалка) или сразу мутация?

## Критерии приёмки
- [ ] Bulk-режим активируется кнопкой «Выбрать», чекбоксы появляются.
- [ ] Удаление: один `POST /trpc/contact.deleteContacts` с массивом, контакты исчезли из списка.
- [ ] Bulk-update: один `POST /trpc/contact.bulkUpdate` с массивом + patch'ем.
- [ ] Ошибка — toast, режим bulk не сбрасывается.
