# US-17 · Добавить Telegram-аккаунт для рассылок

**Предусловие**: аутентифицирован, внутри воркспейса. Аккаунт нужен **именно для outreach-рассылок** — не путать с [US-9](./US-9.md) (личный TG для CRM-чатов).

## Цель
Подключить Telegram-аккаунт, через который будут отправляться рассылочные сообщения кампаний.

## Точка входа
Сайдбар → **«Рассылки»** → **«Telegram-аккаунты»** → **«Добавить Telegram-аккаунт»** → `/w/{id}/outreach/telegram-accounts/new`.

## Флоу
1. **«Номер телефона»** → выбрать **«Страна прокси»** (`Авто` / конкретный код, напр. `KZ`, `RU`) → **«Отправить код»**.
2. Ввести код из Telegram.
3. (Если у аккаунта 2FA): пароль. См. [OQ-1](#открытые-вопросы).
4. Успех: **«Аккаунт успешно добавлен»** → кнопка **«Просмотреть аккаунт»** ведёт в [US-18](./US-18.md).

## Монетизация
На экране — блок **«Нужны дополнительные аккаунты для рассылок?»** с CTA **«Купить Telegram-аккаунты напрямую через нас — $25 / аккаунт»**. Вне scope как отдельная стори.

## Что вызывается
- `proxy.getCountries` — список стран для выпадашки.
- `proxy.getProxyStatus` — статус прокси текущего воркспейса.
- `telegram.account.auth` — отправка кода / submit кода / submit пароля (полиморфная ручка по `step`).
- `telegram.account.authState` — поллинг состояния авторизации между шагами.

Сигнатуры — см. [`api-contracts.md`](./api-contracts.md).

## Success-flow
1. `auth` дошёл до финального шага → аккаунт создан.
2. Редирект на `/w/{id}/outreach/telegram-accounts/{accountId}` (см. [US-18](./US-18.md)).

## Error-flow
- Неправильный код / пароль → `toast.error(err.message)`, текущая форма остаётся.
- Flood-limit от Telegram → отдельный код ошибки (формат — см. [OQ-2](#открытые-вопросы)).

## Открытые вопросы
- **OQ-1**: шаг 2FA-пароля — отдельный `step` в `telegram.account.auth` или отдельная ручка? В capture видим единую `auth`, но без 2FA-сценария.
- **OQ-2**: точные коды ошибок (`PHONE_CODE_INVALID`, `FLOOD_WAIT`, `PHONE_NUMBER_BANNED` и т.д.) — нужен негативный capture.
- **OQ-3**: лимит по кол-ву аккаунтов на воркспейс — если есть, как показывается в UI?

## Критерии приёмки
- [ ] Дропдаун стран заполняется из `proxy.getCountries`.
- [ ] «Отправить код» дёргает `telegram.account.auth` (step=`sendCode`) и переключает форму на ввод кода.
- [ ] Ввод кода → успех → редирект на карточку аккаунта.
- [ ] 2FA-ветка (если включена): форма пароля появляется, submit проходит.
- [ ] Ошибки показываются toast'ом, введённые данные не теряются.
