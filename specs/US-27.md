# US-27 · Глобальное расписание отправок

**Предусловие**: аутентифицирован, внутри воркспейса.

## Цель
Задать часы, в которые outreach-кампании воркспейса могут отправлять сообщения. Применяется ко всем кампаниям сразу.

## Точка входа
Сайдбар → **«Рассылки»** → **«Кампании»** (индекс) → кнопка **«Расписание отправки»** → `/w/{id}/outreach/schedule`.

## Форма
- Матрица **дни недели × часы**.
- **«Таймзона»** — dropdown (точный идентификатор в input — IANA, напр. `Europe/Moscow`).
- Submit: **«Сохранить»**.

## Что вызывается
- `outreach.rescheduleSequences` — input содержит новое расписание + timezone; сервер пересчитывает запланированные отправки во всех кампаниях воркспейса.

Сигнатура — см. [`api-contracts.md`](./api-contracts.md).

## Success-flow
- Toast об успешном обновлении (текст — [OQ-1](#открытые-вопросы)).
- Новые расчёты видны на табе «Обработка» каждой кампании (см. [US-26](./US-26.md)).

## Error-flow
`toast.error(err.message)`, форма остаётся.

## Открытые вопросы
- **OQ-1**: точный текст success-toast.
- **OQ-2**: формат расписания в input — массив `{ day, startHour, endHour }` или bitmap по часам недели?
- **OQ-3**: что если расписание пустое (0 часов в неделю) — UI отклоняет или сервер?

## Критерии приёмки
- [ ] Матрица дней/часов позволяет включать/выключать интервалы.
- [ ] Смена таймзоны применяется ко всем кампаниям.
- [ ] Save → один `POST /trpc/outreach.rescheduleSequences`.
