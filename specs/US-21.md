# US-21 · Создать кампанию из CSV

**Предусловие**: подключён хотя бы один outreach-аккаунт (см. [US-17](./US-17.md)).

## Цель
Загрузить CSV со списком лидов (username / телефон) и создать на его основе кампанию.

## Точка входа
Сайдбар → **«Рассылки»** → **«Кампании»** → **«Новая кампания»** → **«Загрузить CSV»** (подпись **«Загрузить CSV файл с лидами»**) → `/w/{id}/outreach/sequences/new/csv`.

## Шаги
1. **«Загрузите CSV файл с вашими лидами»** → **«Выбрать файл»**.
2. **«Сопоставьте поля со столбцами»**:
    - **«Столбец с username»** — dropdown.
    - **«Столбец с телефоном»** — dropdown.
    - Подпись: **«Выберите, где в файле находятся username и телефон. Достаточно указать только одно поле.»**
3. Если в CSV есть колонки без header'а — warning: **«Предупреждение: Столбцы без заголовка были пропущены»**.
4. **«Предпросмотр»** первых строк → **«Продолжить»**.

## Warning'и при рассылке по номерам
- **«Проверка дубликатов недоступна по номерам телефонов»**.
- **«Некоторые получатели могут быть недоступны из-за их настроек приватности»**.

## Что вызывается
1. `outreach.generateUploadSignedUrl` — получить signed URL для upload файла в storage.
2. (Клиент заливает CSV напрямую в storage по signed URL — не tRPC.)
3. `outreach.lists.uploadCsvList` — создать list-сущность на основе залитого файла, передав путь/ссылку + mapping колонок.
4. `outreach.sequences.create` — создать sequence, привязанный к этому list'у.

Сигнатуры — см. [`api-contracts.md`](./api-contracts.md).

## Success-flow
1. Все три вызова прошли → получили `sequenceId`.
2. Редирект на `/w/{id}/outreach/sequences/{sequenceId}` (карточка кампании).

## Error-flow
- Signed URL failed → toast, попытка заново.
- Upload в storage упал → toast.
- `uploadCsvList` отклонил mapping (обязательное поле пропущено) → toast.
- `toast.error(err.message)` как универсальный fallback.

## Открытые вопросы
- **OQ-1**: формат input `uploadCsvList` — передаётся ли там путь файла в storage, или уникальный token из signed URL?
- **OQ-2**: лимиты на размер CSV / количество строк — нужно проверить.
- **OQ-3**: что происходит с дубликатами по username (если есть `username`-колонка) — дедуп автоматически или оставляются как есть?

## Критерии приёмки
- [ ] Upload CSV проходит: `generateUploadSignedUrl` → PUT в storage → `uploadCsvList` в правильном порядке.
- [ ] Mapping-форма отображает все колонки CSV и требует хотя бы одного из `username`/`phone`.
- [ ] Warning'и о дубликатах и приватности видны, если выбран `phone`-столбец.
- [ ] «Продолжить» приводит к `sequences.create` и редиректу на карточку.
