# US-23 · Привязать TG-аккаунты к кампании

**Предусловие**: кампания создана (см. [US-20](./US-20.md)), есть хотя бы один outreach-аккаунт (см. [US-17](./US-17.md)).

## Цель
Выбрать, через какие TG-аккаунты будут отправляться сообщения этой кампании.

## Точка входа
Внутри кампании → таб **«Аккаунты»** → `/w/{id}/outreach/sequences/{sequenceId}/accounts`.

## Форма
Radio-переключатель:
- **«Использовать все аккаунты»** — авто-включение всех существующих и будущих аккаунтов воркспейса.
- **«Использовать выбранные аккаунты»** — под ним список с чекбоксами по каждому аккаунту.

Submit: **«Сохранить»**.

## Что вызывается
- `outreach.sequences.patch` — input включает `accounts: "all"` или `accounts: string[]` (точный формат — см. [OQ-1](#открытые-вопросы)).

Сигнатура — см. [`api-contracts.md`](./api-contracts.md).

## Success-flow
- `patch` прошёл → в checklist ставится галка «Аккаунты».

## Error-flow
`toast.error(err.message)`, выбор не сбрасывается.

## Открытые вопросы
- **OQ-1**: точное имя поля в `sequences.patch` — `accountIds` / `accounts` / `useAllAccounts` + `selectedAccountIds`?
- **OQ-2**: что если в режиме «все» добавился новый аккаунт — кампания его подхватит автоматически? Ожидаем да, но нужен подтверждающий capture.

## Критерии приёмки
- [ ] Radio-переключатель корректно показывает текущее состояние при загрузке.
- [ ] Список чекбоксов появляется только в режиме «выбранные».
- [ ] Save → один `POST /trpc/outreach.sequences.patch`.
