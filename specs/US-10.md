# US-10 · Чаты

> ⚠️ **Firestore-heavy — основной канал данных не capture'нут.** Сообщения, метаданные чатов, unread-счётчики приходят через Firestore subscriptions. tRPC-ручки здесь вторичны (бутстрэп правой панели, schedule calendar event). Полная стори будет написана после прокликанного capture-прохода с Firestore-sniffer.

**Предусловие**: подключён личный TG (см. [US-9](./US-9.md)).

## Цель
Читать и отвечать в своих TG-чатах из CRM, привязывать чат к контакту/лиду, запускать действия над ним.

## Точка входа
Сайдбар → **«Чат»** → `/w/{id}/telegram`.

## Базовые состояния
- **Empty**: **«Выберите чат»** (когда ни один не открыт).
- **Loaded**: список чатов слева + контент справа.

## Что прокликать для полного capture
1. **Открыть чат** — клик по чату в списке. Ожидается: `_firestore.html` снапшот с loaded-перепиской.
2. **Переключиться на другой чат** — контент справа обновится.
3. **Отправить текстовое сообщение** — input → Enter / кнопка отправки.
4. **Получить входящее** — собеседник пишет с другого TG-клиента; сообщение появляется без reload.
5. **Скролл истории вверх** — подгрузка старых (Firestore pagination).
6. **Read receipt** — переключение с непрочитанного чата; счётчики обновляются.
7. **Reply / Forward / Edit / Delete** — если кнопки есть в UI.
8. **Attach / emoji / voice** — если поддерживается.
9. **«Создать лид в CRM»** — правая панель карточки лида (кнопки «Lead Menu», «Редактировать лид», «Добавить описание», «Добавитьзаметку», «Добавитьнапоминание»).
10. **Перейти в карточку контакта** — клик по «Редактировать лид» / имени (см. [US-14](./US-14.md)).
11. **«Открыть в Telegram»** — deep-link в native TG.

## Что вызывается (HTTP)
- `telegram.account.getAccountConnectionData` — бутстрэп.
- `proxy.getProxyStatus` — бутстрэп.
- `activity.scheduleCalendarEventIfPossible` — если из чата создаётся напоминание с датой.

## Основной канал данных
Firestore subscriptions (messages, chat metadata, unread counters).

После прокликивания смотрим `processed/routes/ProtectedWWorkspaceIdTelegramRouteImport/*_firestore.html` для извлечения реальной структуры чата.

## Открытые вопросы
- **OQ-1**: Firestore-пути коллекций `chats`, `messages`, `unread` — формат и вложенность.
- **OQ-2**: как отправка сообщения уходит в Telegram — через Firestore-trigger на backend'е или есть отдельный RPC (`telegram.client.sendMessage`)?
- **OQ-3**: существуют ли Reply / Forward / Edit / Delete в UI?
- **OQ-4**: есть ли support вложений и как они загружаются (signed URL?).
