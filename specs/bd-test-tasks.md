- [x] **Убрать «создано каналов N»** из модалки добавления каналов — пользователю

- [x] **Баг: каналы сами уходят из инбокса «нет контактов» в «в работе» без контакта.**
  Исчезали без кликов оператора, всплывали как «в работе» с пустым контактом.

  **Причина (подтверждена Codex по prod-БД + код):** статус не хранится, считается
  на лету. Предикат «контакт готов» (`contact-sql.ts`) засчитывал готовым любой
  канал, у которого просто ЕСТЬ бесплатная личка (`direct_messages_chat_id` из
  sync). А открытие карточки канала автоматически (без клика, TTL 24ч) запускало
  Telegram-sync, который эту личку и находил → лид молча становился «готов» →
  вылетал из «нет контактов» в «в работе». Отправки при этом нет (ничего не
  планирует) → лид в лимбо. Каскад: список перестраивается → открывается следующая
  карточка → её sync → опять (отсюда «список прыгает», скачок счётчика без кликов).

  **Фикс (сделано):** убрал авто-ветку «есть бесплатная личка» из `contactReadySql`.
  Теперь готовность держится только на явном выборе оператора: @username админа,
  привязанный контакт, или явно выбранный способ связи (`contact_method.kind`,
  включая «Использовать личку канала»). Простой просмотр карточки больше ничего не
  переклассифицирует → каскад/прыжки уходят вместе с багом. Файлы: `contact-sql.ts`,
  комментарий в `placement-recipient.ts`.

  **Осталось отдельной задачей (НЕ багфикс):** клик «Использовать личку канала»
  сейчас делает лид готовым, но НЕ ставит в очередь авто-рассылки — планировщик
  требует `username`, воркер шлёт только живому tg-юзеру. Авто-отправка опенера в
  личку канала в пайплайне не построена (только ручная отправка через чат лички).
  Если нужна авто-рассылка в личку — это фича: добавить DM-таргет в планировщик
  (`scheduleUnscheduledLeads`/`scheduleDolivkaForChannel`) и в воркер.

  **Отдельный смежный риск (не этот баг, на будущее):** входящее сообщение ставит
  `repliedAt` на ВСЕ `projectItems` с данным `tgUserId` по всему воркспейсу
  (`outreach-listener.ts`), а `repliedAt` проверяется раньше `no_contact`. Может
  флипать лиды, у которых висит `tgUserId` без полноценного контакта. Пока не
  воспроизведено в этой сессии — держим на радаре.
