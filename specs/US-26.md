# US-26 · Запуск, пауза, статистика, удаление кампании

**Предусловие**: кампания создана и настроена — сообщения (US-22), аккаунты (US-23), contact-settings (US-24).

## Цель
Управлять жизненным циклом кампании: запускать, ставить на паузу, смотреть метрики, удалять.

## Точка входа
Внутри кампании → таб **«Детали кампании»** → `/w/{id}/outreach/sequences/{sequenceId}`.

## Статусы
- **`Черновик`** — создана, ещё не запущена.
- **`Активна`** — запущена, сообщения отправляются по расписанию.
- **`На паузе`** — временно приостановлена.

## Действия в «Деталях кампании»
- **«Запустить»** — `Черновик` / `На паузе` → `Активна`.
- **«Приостановить»** — `Активна` → `На паузе`.
- **«Удалить кампанию»** — удаление (подтверждение ожидаем, [OQ-1](#открытые-вопросы)).

## Вкладки
### «Обработка»
Подпись: **«Отслеживайте прогресс · Здесь вы можете увидеть все запланированные сообщения, а также какие-либо ошибки»**.
Подсказки: **«Перезапуск кампании часто помогает при любой ошибке»**, **«Вернитесь в раздел Рассылки и проверьте Расписание кампаний»** (см. [US-27](./US-27.md)).

### «Статистика»
Метрики: **«Отправлено»** / **«Прочитали»** / **«Ответили»** → дёргает `outreach.getSequenceStats`.

### «Аналитика»
- **«Период»**: `7 дней` / `30 дней` / `90 дней` / `Свой` (календарь-пикер с диапазоном).
- Группировка: `По дням` / `По неделям` / `По месяцам`.
- **«Отображать»**: `По дате отправки` / `По дате события`.
→ `outreach.getSequenceAnalytics`.

## Что вызывается
- `outreach.updateSequenceStatus` — смена статуса (start / pause).
- `outreach.rescheduleSequences` — дёргается при start (если нужно пересчитать расписание).
- `outreach.getSequenceStats` — метрики для таба «Статистика».
- `outreach.getSequenceAnalytics` — для таба «Аналитика».
- `outreach.sequences.delete` — удаление.

Сигнатуры — см. [`api-contracts.md`](./api-contracts.md).

## Success-flow
- Start: статус меняется на `Активна`, на processing-табе появляются запланированные отправки.
- Pause: статус `На паузе`, новые отправки не планируются.
- Delete: редирект на `/w/{id}/outreach/sequences` (список), кампания пропала.

## Error-flow
`toast.error(err.message)`.

## Открытые вопросы
- **OQ-1**: confirm-диалог на «Удалить» — ожидаем, но в capture не зафиксирован.
- **OQ-2**: что происходит при start кампании с 0 лидами — отклоняется или стартует вхолостую?
- **OQ-3**: `rescheduleSequences` — с каким scope дёргается: все кампании воркспейса или только текущая?

## Критерии приёмки
- [ ] Start/pause меняют статус, дёргают `updateSequenceStatus`.
- [ ] Табы «Обработка» / «Статистика» / «Аналитика» корректно грузят свои данные.
- [ ] Фильтры аналитики (период, группировка, ось) меняют параметры `getSequenceAnalytics`.
- [ ] Delete удаляет кампанию и уводит со страницы.
