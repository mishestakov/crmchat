# US-11 · Список контактов

**Предусловие**: аутентифицирован, внутри воркспейса.

## Цель
Посмотреть всех контактов (лидов) воркспейса, отфильтровать/отсортировать, сохранить представление, переключиться в режим воронки.

## Точка входа
Сайдбар → **«Лиды»** → `/w/{id}/contacts`.

Два режима отображения:
- **«Список»** (таблица).
- **«Воронка»** (kanban по свойству `Стадия`).

## Элементы UI (общие для обоих режимов)
- **«Поиск»** — поле поиска + кнопка **«Очистить»**.
- **«Выбрать»** — bulk-режим (чекбоксы по строкам, см. [US-16](./US-16.md)).
- **«Фильтр»** — добавление условий по property.
- **«Настройки представления»** → раскрывает: **«Сортировка: По умолчанию»**, **«Столбцы»**, **«Скрыть пустые столбцы»**, **«Сбросить»**.
- **«Новый лид»** — ведёт в [US-12](./US-12.md).

## Pipeline view
Kanban по property **«Стадия»** с колонками из CRM-воронки:
**`Лид` → `Разговор` → `Предложение` → `Переговоры` → `Сделка закрыта`**.

Эти же названия дублируются в сайдбаре → «Воронка» как shortcut'ы.

### Empty pipeline state
Если property **«Стадия»** не настроено:
> «Let's setup a pipeline / Start by creating a property to define your pipeline stages / Create property»

Кнопка ведёт в [US-7](./US-7.md) (создать property типа `single-select`).

## Empty contacts state
Если в воркспейсе ноль контактов:
> «Создайте свой первый лид / Синхронизируйте папки Telegram / Или добавьте вручную»

Две кнопки — ссылка на [US-9](./US-9.md) и на [US-12](./US-12.md) соответственно.

## Save view
1. Кнопка **«Сохранить представление для всех»** → модалка **«Вы хотите обновить текущее представление или создать новое?»**.
2. Ветка **«Обновить представление»** — перезаписывает текущее.
3. Ветка **«Сохранить как новое представление»** → форма **«Название представления»** + кнопки **«Отменить»** / **«Создать представление»**.
4. Toast: **«Представление сохранено»**.

Сами представления (views) хранятся в Firestore — tRPC-вызовов create/update не зафиксировано. См. [OQ-1](#открытые-вопросы).

## Что вызывается
- `workspaces.getMembers` — бутстрэп (нужно для owner-пикера, аватарок в строках).
- `contact.bulkUpdate` — массовое изменение свойств (используется в [US-16](./US-16.md)).
- `contact.deleteContacts` — массовое удаление (также US-16).

Список самих контактов в capture виден как Firestore-подписка, а не tRPC. См. [OQ-2](#открытые-вопросы).
