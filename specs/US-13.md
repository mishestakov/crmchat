# US-13 · Создать контакт из QR

> ⚠️ **Не в полном capture.** Ручка `contact.createContactFromQr` объявлена в code-inventory, но в сессии capture ни роут (модалка QR), ни сам вызов не прошли. Эту стори нужно перекрыть отдельным capture-сетом (сканировать/вставить QR из Telegram). Ниже — план.

## Цель
Создать контакт, отсканировав Telegram-QR другого пользователя (например, QR из «Мой Telegram-QR»).

## Точка входа
Сайдбар → **«Лиды»** → **«Список»** → **«Новый лид»** → **«Сканировать QR»** (см. [US-12](./US-12.md)).

## Что вызывается (declared)
- `contact.createContactFromQr` — input содержит QR-данные, output — созданный контакт. Точная сигнатура — см. [`api-contracts.md`](./api-contracts.md) (declared-but-not-captured).

## Ожидаемый флоу (не подтверждено capture)
1. Модалка **«Выберите, как создать новый лид»** → клик **«Сканировать QR»**.
2. Открывается sub-шаг с QR-сканером. Предполагаемые варианты UI:
   - Камера браузера (getUserMedia + QR-decode) — ожидаемо для web.
   - Загрузить картинку с QR.
   - Paste raw-текст/base64 QR-данных.
3. После распознавания → `contact.createContactFromQr({ qr: ... })`.
4. Success → редирект на `/contacts/{newContactId}` (см. [US-14](./US-14.md)).

## Что прокликать для capture
1. Открыть Лиды → Новый лид → «Сканировать QR» — снять скриншот модалки.
2. Загрузить / вставить QR «Мой Telegram-QR» другого пользователя.
3. Снять request/response `contact.createContactFromQr` и финальный URL.
4. Повторить с QR уже существующего контакта, чтобы увидеть поведение дубликата.

## Открытые вопросы
- **OQ-1**: формат input `contact.createContactFromQr` — raw text / base64 / JSON-payload от Telegram?
- **OQ-2**: UI модалки: камера / upload / paste — какие варианты реально есть?
- **OQ-3**: QR уже существующего контакта → дубль, merge или ошибка с ссылкой на существующий?
- **OQ-4**: что если QR не-Telegram (чужой формат) — какая ошибка?
