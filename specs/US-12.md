# US-12 · Создать лида

**Предусловие**: аутентифицирован, внутри воркспейса.

## Цель
Добавить новый контакт (лид) одним из трёх способов.

## Точка входа
Сайдбар → **«Лиды»** → **«Список»** → **«Новый лид»** → модалка **«Выберите, как создать новый лид»** → `/w/{id}/contacts/new`.

## Три способа

### A. «Форвадни сообщение»
Переслать сообщение в TG-бот → контакт подтягивается автоматически. Подробности внутреннего флоу — вне UI. См. [OQ-1](#открытые-вопросы).

### B. «Сканировать QR»
Ведёт в [US-13](./US-13.md).

### C. «Добавить вручную» — форма
| Поле | Label | Плейсхолдер / Примечание |
|---|---|---|
| `name` | **«Имя»** | обязательное |
| `telegramUsername` | **«Имя пользователя Telegram»** | `например, durov` |
| `shortDescription` | **«Краткое описание»** | — |
| `description` | **«Описание»** | многострочное |
| `stage` | **«Стадия»** | dropdown из property-воронки |
| `url` | **«URL»** | — |
| `email` | **«Email»** | — |
| `phone` | **«Телефон»** | — |

Ниже системных полей — кастомные поля воркспейса + кнопка **«Новое поле»** (ведёт в [US-7](./US-7.md)).

## Что вызывается
В capture запрос создания контакта **не зафиксирован** — запись идёт напрямую в Firestore (`contacts/{workspaceId}/items`). См. [OQ-2](#открытые-вопросы).

## Success-flow
Переход на `/contacts/{contactId}` — карточка нового контакта (см. [US-14](./US-14.md)).

## Открытые вопросы
- **OQ-1**: как работает «Форвадни сообщение» — есть ли polling, или обновление приходит через Firestore?
- **OQ-2**: точная Firestore-коллекция и формат документа `contact`. Нужен capture с Firestore-sniffer.

## Критерии приёмки
- [ ] Модалка предлагает три способа, каждая кнопка ведёт в нужный поток.
- [ ] Форма (ручной способ): пустое `name` блокирует submit.
- [ ] После создания — редирект на карточку контакта.
