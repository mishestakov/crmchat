# US-19 · Переместить TG-аккаунты

> ⚠️ **Bulk-actions UI не прокликан в capture.** Ручка `telegram.account.moveAccounts` captured, но сам клик-путь и форма выбора target-воркспейса — не зафиксированы. Часть полей ниже — из названий RPC / code-inventory.

**Предусловие**: есть минимум один TG-аккаунт (см. [US-17](./US-17.md)) и минимум два воркспейса (см. [US-5](./US-5.md)).

## Цель
Переместить один или несколько outreach-аккаунтов в другой воркспейс.

## Точка входа
Сайдбар → **«Рассылки»** → **«Telegram-аккаунты»** → **«Выбрать»** (bulk-режим) → выделить чекбоксами → **«Переместить»**.

## Форма
Ожидаем модалку с выбором target-воркспейса (из тех, в которых пользователь состоит). См. [OQ-1](#открытые-вопросы).

## Что вызывается
- `telegram.account.moveAccounts` — input `{ accountIds, targetWorkspaceId }` (формат — нужен capture, см. [OQ-2](#открытые-вопросы)).

## Success-flow
1. Аккаунты исчезают из списка текущего воркспейса.
2. Toast об успехе (текст — [OQ-3](#открытые-вопросы)).

## Error-flow
`toast.error(err.message)`.

## Открытые вопросы
- **OQ-1**: UI-форма — dropdown / модалка / отдельная страница? Прокликать и снять скриншот.
- **OQ-2**: точный input `moveAccounts` — `targetWorkspaceId`, требуется ли также `sourceWorkspaceId`?
- **OQ-3**: текст success-toast.
- **OQ-4**: что происходит с кампаниями, которые использовали эти аккаунты? Они остаются в исходном воркспейсе с «сиротскими» аккаунтами или аккаунты отвязываются?

## Ожидаемый флоу (не подтверждено capture)
1. На `/outreach/telegram-accounts` → **«Выбрать»** переводит список в bulk-режим (чекбоксы у каждого аккаунта).
2. Панель bulk-actions появляется внизу/сверху → кнопка **«Переместить»**.
3. Модалка: dropdown **«Рабочее пространство»** со списком воркспейсов пользователя, где он `admin`/`owner` (по аналогии с US-4 permissions).
4. Submit → `telegram.account.moveAccounts({ workspaceId: source, targetWorkspaceId, accountIds })`.
5. Success → аккаунты исчезают из текущего списка, toast.

## Что прокликать для capture
1. Завести минимум два воркспейса (см. [US-5 · 5.1](./US-5.md)) и подключить 2+ аккаунта (см. [US-17](./US-17.md)).
2. Зайти в список аккаунтов → «Выбрать» → отметить 1-2.
3. «Переместить» → выбрать другой воркспейс → capture request/response.
4. Проверить побочные эффекты в старом воркспейсе (кампании, использовавшие эти аккаунты).

## Критерии приёмки (финализировать после capture)
- [ ] Bulk-режим активируется и закрывается корректно.
- [ ] Модалка показывает только те target-воркспейсы, в которые у пользователя есть доступ.
- [ ] Один `POST /trpc/telegram.account.moveAccounts` на все выбранные аккаунты.
- [ ] После успеха аккаунты видны в target-воркспейсе, пропали из source.
