# US-20 · Создать кампанию из CRM (с фильтрами)

**Предусловие**: настроены property (см. [US-7](./US-7.md)), есть контакты (см. [US-11](./US-11.md)), подключён хотя бы один outreach-аккаунт (см. [US-17](./US-17.md)).

## Цель
Создать рассылочную кампанию, выбрав получателей из контактов CRM по фильтру.

## Точка входа
Сайдбар → **«Рассылки»** → **«Кампании»** → **«Новая кампания»** → `/w/{id}/outreach/sequences/new`.

На этом экране — выбор источника. Для CRM-ветки: **«Лиды из CRM»** (подпись **«Использовать лидов из CRM»**) → `/w/{id}/outreach/sequences/new/crm`.

## Шаги CRM-ветки
1. **Тип кампании**:
    - **«Одноразовая»** — создать один раз, получатели фиксируются в момент создания.
    - **«Динамическая»** — подпись: «Все лиды из CRM, которые удовлетворяют фильтрам, будут автоматически добавляться в эту кампанию».
2. **Фильтры** → **«Добавить фильтр»** → условие `property == value` (например `Стадия: Разговор`). Несколько фильтров.
3. **«Предварительный просмотр»** — показывает кол-во лидов, соответствующих фильтру.
    - Warning при 0 совпадений: **«Нет лидов, соответствующих вашим фильтрам. Здесь отображаются только лиды с именем пользователя Telegram»**.
4. **«Создать»** — попадает в черновик кампании → редирект на карточку (US-22..US-26).

## Что вызывается
- `outreach.sequences.create` — создание sequence-документа. Input содержит `type` (one-shot / dynamic), `source: "crm"`, фильтры, name.
- Создание CRM-list (фильтр-набор получателей) — declared `outreach.lists.createCrmList`, но в capture отсутствует; возможно, он вшит в `sequences.create` или дёргается позже. См. [OQ-1](#открытые-вопросы).

Сигнатуры — см. [`api-contracts.md`](./api-contracts.md).

## Success-flow
1. `sequences.create` → получили `sequenceId`.
2. Редирект на `/w/{id}/outreach/sequences/{sequenceId}` (таб «Детали кампании»).

## Error-flow
`toast.error(err.message)`, форма остаётся.

## Открытые вопросы
- **OQ-1**: когда именно дёргается `outreach.lists.createCrmList`? При клике «Предварительный просмотр» или вшит в `sequences.create`?
- **OQ-2**: формат фильтров в input — JSON-дерево (AND/OR) или плоский список? Нужен capture с несколькими фильтрами.
- **OQ-3**: разница в input `sequences.create` между one-shot и dynamic — какие доп. поля у dynamic?

## Критерии приёмки
- [ ] Выбор источника работает, CRM-ветка открывается.
- [ ] Фильтры добавляются и убираются, «Предпросмотр» показывает корректное кол-во.
- [ ] Warning о 0 совпадениях виден для пустой выборки.
- [ ] «Создать» → один `POST /trpc/outreach.sequences.create`, редирект на карточку sequence.
