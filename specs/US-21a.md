# US-21a · Кампания из «Группы из CRM»

> ⚠️ **Нет capture.** Ветка не прокликана: UI формы, используемые ручки и семантика «групп» неизвестны.

**Предусловие**: та же, что у [US-20](./US-20.md) / [US-21](./US-21.md).

## Точка входа (из user-stories.md)
Сайдбар → **«Рассылки»** → **«Кампании»** → **«Новая кампания»** → **«Группы из CRM»** (подпись **«Отправить рассылку по группам»**).

## Ожидаемый флоу (не подтверждено capture)

Две правдоподобные интерпретации «группы»:

**Гипотеза A** — Telegram-группа/канал, участников которой подтягиваем в лиды.
1. Выбрать TG-группу из списка доступных (через `telegram.client.*` ручки личного синка).
2. Система перечисляет участников → создаёт лидов в CRM → создаёт кампанию.
3. Ожидаемые ручки: `telegram.client.getChats` или аналог → `outreach.lists.createFromChatMembers` (имя гипотетическое).

**Гипотеза B** — «группа» = сохранённый сегмент контактов CRM (тег/представление).
1. Список сегментов → выбрать один → превью → создать кампанию.
2. Ожидаемые ручки: `outreach.lists.createFromSegment` / `sequences.create` с `source: "segment"`.

## Что прокликать для capture
1. Открыть `/outreach/sequences/new` → кликнуть **«Группы из CRM»**.
2. Снять URL, скриншот формы, перечень доступных «групп».
3. Создать кампанию до конца и посмотреть все network-вызовы.

## Открытые вопросы
- **OQ-1**: что такое «группа» — TG-группа или CRM-сегмент (см. гипотезы выше)?
- **OQ-2**: какой URL у этого sub-роута? Скорее всего `/outreach/sequences/new/groups` или `/groups-crm`.
- **OQ-3**: какие ручки реально дёргаются? Свериться с `outreach.lists.*` и `outreach.sequences.create` input'ами.

## Критерии приёмки (финализировать после capture)
- [ ] Страница/модалка **«Группы из CRM»** открывается из меню «Новая кампания».
- [ ] Список групп/сегментов загружается, превью показывает кол-во лидов.
- [ ] Create → `outreach.sequences.create` с корректным `source` + ссылкой на list.
