# US-25 · Управление лидами кампании

**Предусловие**: кампания создана и в ней есть лиды (см. [US-20](./US-20.md) / [US-21](./US-21.md)).

## Цель
Посмотреть всех лидов кампании в таблице, вручную вмешаться в их судьбу: убрать, отправить сейчас, обновить property, разрешить дубликаты.

## Точка входа
Внутри кампании → таб с лидами → `/w/{id}/outreach/sequences/{sequenceId}/leads`.

## Таблица
Колонки: **«Лид»** / **«Аккаунт»** / **«Telegram»** / **«Действия»**.

Тоггл **«Показать CSV данные»** разворачивает дополнительные колонки (если кампания из CSV):
`first_name`, `last_name`, `email`, `company`, `position`, `city`, `country`, `industry`, `deal_size`, `lead_source`, `interest_level`, `notes`.

> Ожидаем, что для CRM-кампаний toggle либо скрыт, либо разворачивает кастомные property контакта.

## Действия (колонка «Действия» / per-row)
- **«Убрать из кампании»** → `outreach.removeLeadFromSequence`.
- **«Отправить сейчас»** → форсированная отправка первого сообщения (имя ручки — см. [OQ-1](#открытые-вопросы); declared `sendOutreachMessageNow`).
- **«Обновить property»** → `outreach.updateLeadProperties`.
- **«Resolve duplicates»** → `outreach.resolveDuplicates` (если у лида есть дубликаты в CRM).

## Что вызывается
- `outreach.getLeads` — load списка (пагинация — см. [OQ-2](#открытые-вопросы)).
- `outreach.removeLeadFromSequence` — убрать конкретного лида.
- `outreach.updateLeadProperties` — обновить property у лида (без выхода из кампании).
- `outreach.resolveDuplicates` — merge / разрешить дубликатов.

Сигнатуры — см. [`api-contracts.md`](./api-contracts.md).

## Error-flow
`toast.error(err.message)`. После ошибки состояние таблицы не откатывается автоматически — на `getLeads` refetch.

## Открытые вопросы
- **OQ-1**: declared `sendOutreachMessageNow` — ручка есть, но в capture вызова нет. Нужен прокликанный сценарий.
- **OQ-2**: пагинация `getLeads` — offset-based или cursor-based? Размер страницы.
- **OQ-3**: UI-форма «Обновить property» — модалка с выбором property + value? Прокликать.

## Критерии приёмки
- [ ] Таблица загружается через `outreach.getLeads`.
- [ ] Toggle «Показать CSV данные» доступен для CSV-кампаний.
- [ ] Каждое per-row действие дёргает ровно одну ручку.
- [ ] После действия таблица обновляется (refetch или оптимистический апдейт).
