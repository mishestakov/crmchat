# US-22 · Настроить сообщения кампании

**Предусловие**: кампания создана (см. [US-20](./US-20.md) / [US-21](./US-21.md)).

## Цель
Добавить в кампанию одно или несколько сообщений, настроить задержки между ними, подставить переменные, посмотреть превью на реальном лиде.

## Точка входа
Сайдбар → **«Рассылки»** → **«Кампании»** → клик по кампании → таб **«Детали кампании»** → **«Outreach checklist»** → **«Добавить ваше первое сообщение»**.

## Редактор сообщения
- Текстовое поле с форматированием: **`Format bold`** / **`Format italic`**.
- **«Insert variable»** — typeahead-меню переменных (пример: `​Username​`). Перечень переменных и их ключи — см. [OQ-1](#открытые-вопросы).

## Цепочка сообщений
- **«Добавить сообщение»** — следующее сообщение в цепочке.
- Для второго и далее — поле **«Через N день отправить текст»** (задержка).

## Превью
- **«Предпросмотр сообщения»** → **«Предпросмотр для лида»** → **«Случайный»** (подставляет реального лида, напр. `@mikeshestakov_dev`) → **«Закрыть»**.

## Что вызывается
- `outreach.validateTextVariables` — валидация текста на незакрытые/несуществующие переменные (ожидаем дёргается на каждый blur / submit).
- `outreach.sequences.patch` — сохранение тела кампании (массив сообщений, задержки).

Сигнатуры — см. [`api-contracts.md`](./api-contracts.md).

## Success-flow
- Save (implicit / explicit) → `sequences.patch` → данные сохранены.
- В checklist галка «Сообщения» ставится.

## Error-flow
- `validateTextVariables` вернул ошибку → inline-highlight проблемного места в тексте (см. [OQ-2](#открытые-вопросы)).
- `toast.error(err.message)` на общий fallback.

## Открытые вопросы
- **OQ-1**: полный список переменных в «Insert variable» — нужен capture с раскрытым меню.
- **OQ-2**: формат ошибок `validateTextVariables` — пара `{ line, col, message }` или просто текстовое сообщение?
- **OQ-3**: есть ли вложения (фото, документы) в сообщениях? В user-stories.md упомянут `generateUploadSignedUrl` как кандидат, но не для этого экрана.
- **OQ-4**: максимум сообщений в цепочке и максимум задержки.

## Критерии приёмки
- [ ] Добавление первого сообщения доступно из checklist.
- [ ] Каждый save дёргает `outreach.sequences.patch` (batch ок).
- [ ] Переменные из typeahead вставляются корректно.
- [ ] Превью подставляет реальные значения, «Случайный» обновляет лида.
- [ ] Ошибка валидации переменной не даёт сохранить.
