# User Stories — carte из сценариев

Цель документа: пройти сервис **осмысленно**, а не дёргая роуты. Каждая история = один пользовательский сюжет. Покрытие: все 38 keep-маршрутов + ~56 RPC из `scope.json`.

Формат: **Открыть** (куда кликнуть в меню) → **Шаги** (что делать на экране) → **RPC** (что ждём в capture) → **Варианты** (ветвления).

URL-ы указаны справочно в скобках; основной способ входа — через меню/клики.

---

## Карта сайдбара (realные лейблы из DOM)

Левый сайдбар (сверху вниз):
- **Лиды** — раскрывается в подпункты:
  - **Воронка** (kanban, стадии `Лид / Разговор / Предложение / Переговоры / Сделка закрыта`)
  - **Список** (table view) → `/w/{id}/contacts`
- **Чат** → `/w/{id}/telegram`
- **Рассылки** — раскрывается:
  - **Кампании** → `/w/{id}/outreach`
  - **Telegram-аккаунты** → `/w/{id}/outreach/telegram-accounts`
- **Настройки** → `/w/{id}/settings` (хаб всех вложенных настроек)
- **Кастомные поля** → `/w/{id}/settings/properties/contacts`
- **Команда** → `/w/{id}/settings/workspace` (список участников)
- **Интеграции** — вне scope (drop)
- **Инструменты** (вне scope, drop): AI Bot, Парсер групп, Web3 Database, Синк ТГ-каналов, Синк ТГ-папок, Телефоны → Telegram
- **Поменять тему**, **Помощь**, **Заработай 500 $** — низ сайдбара, вне scope

Большинство вложенных экранов (кампания → Аккаунты / Лиды / Настройки / Обработка / Статистика / Аналитика) — через **табы внутри кампании**, не через сайдбар.

## Этап 1 — Воркспейс

### US-1 · Первый вход и создание воркспейса
- **Открыть**: корневой `/` после логина → если у пользователя нет воркспейса, редиректит на форму создания
- **Шаги**: ввести «Название рабочего пространства» → «Создать рабочее пространство»
- **RPC**: `workspace.createWorkspace` · `telegram.authenticateByInitData` (если вход через TG) · `workspaces.getMembers`
- *(URL: `/settings/workspace/new`)*

### US-5 · Управление воркспейсами
Три подсюжета одного экрана «Команда»/«Настройки»:

- **Создать ещё один**: сайдбар → «Настройки» → «Создать рабочее пространство» → имя → «Создать» → переключиться через picker наверху сайдбара. RPC: `workspace.createWorkspace`. URL: `/settings/workspace/new`.
- **Переименовать**: сайдбар → «Команда» → inline-поле «Название рабочего пространства» → «Save». Toast: «Название рабочего пространства сохранено!». RPC: _(не в scope — вероятно Firestore-write; проверить при клике)_.
- **Настройки организации**: путь через UI **не найден** — возможно только прямой URL `/w/{id}/settings/organization/{organizationId}`. Inline-поле «Название организации» → «Сохранить». RPC: `workspaces.getMembers` (остальное Firestore). **OQ**: как создаётся организация и как к ней привязываются воркспейсы?

## Этап 2 — Команда

Все стори этого этапа стартуют с **сайдбар → «Команда»** = `/w/{id}/settings/workspace`. На этой странице виден список членов + раздел «Ожидающие приглашения» + кнопка «Пригласить члена команды».

### US-2 · Пригласить коллегу
- **Открыть**: сайдбар → «Команда» → кнопка «Пригласить члена команды»
- **Шаги**: ввести «Имя пользователя Telegram», выбрать роль (**`Участник` / `Админ` / `Чаттер`**) → «Пригласить»
- **Privacy-warning в UI**: «⚠️ Члены команды в этом рабочем пространстве увидят ваши чаты Telegram»
- **Toast при успехе**: «Пользователь успешно приглашен»
- **RPC**: `workspace.inviteWorkspaceMember` · `workspace.getPendingInvites` · `workspaces.getMembers`
- *(URL: `/settings/workspace/invite`)*

### US-3 · Принять приглашение
- **Открыть**: по ссылке, которую прислал бот (не через меню). Два варианта:
  - не залогинен: `/accept-invite/{workspaceId}/{inviteCode}`
  - залогинен под другим воркспейсом: `/w/{curId}/settings/workspace/accept-invite/{wId}/{inviteCode}`
- **Шаги**: подтвердить → редирект в принятый воркспейс
- **RPC**: `workspace.getWorkspaceInvite` · `workspace.acceptWorkspaceInvite`

### US-4 · Управлять участником
- **Открыть**: сайдбар → «Команда» → клик по участнику в списке
- **Шаги**: сменить «Роль» через дропдаун / «Покинуть рабочее пространство» (для себя) / «Удалить» (для других)
- **Ограничение в UI**: «Вы не можете изменить свою собственную роль»
- **Поля карточки**: «Часовой пояс» (напр. `Europe/Moscow`), «Роль»
- **Ожидающие приглашения**: раздел «Ожидающие приглашения», у каждого `осталось N days`
- **RPC**: `workspace.changeWorkspaceMemberRole` · `workspace.removeWorkspaceMember` · `workspaces.getMembers`
- *(URL: `/settings/workspace/user/{userId}`)*

## Этап 3 — Структура данных (Properties)

### US-6 · Посмотреть свойства
- **Открыть**: сайдбар → «Кастомные поля»
- **Шаги**: просмотреть список. Подсказка: «Вы можете перетаскивать элементы для изменения порядка полей», ссылка «Как использовать поля?»
- **`{objectType}` в URL**: `contacts` (единственный наблюдавшийся). Проверить есть ли другие типы объектов в UI (сейчас нет)

### US-7 · Создать кастомное поле
- **Открыть**: сайдбар → «Кастомные поля» → «Новое поле» → выбрать тип
- **Шаги**: «Название поля», «Изменить цвет», «Тип поля», «Обязательное поле», «Отображать в списке» → «Создать поле»
- **Варианты типа**:
  - Text (обычное текстовое)
  - **«Одиночный выбор»** (`single-select` в URL)
  - **«Множественный выбор»** (`multi-select` в URL)
  - _Из описания_: числа / даты / прочее. Прокликать если UI предлагает — прижать через capture
- **Для select-типов**: блок «Значения» — список опций, «Новое значение», у каждой «Изменить цвет» (палитра: Без цвета / Серый / Коричневый / Оранжевый / Желтый / Зеленый / Синий / Фиолетовый / Розовый / Красный)
- **RPC**: `workspaces.getMembers` (shell); создание — Firestore
- **Ключ поля**: `custom.<id>`

### US-8 · Редактировать/удалить поле
- **Открыть**: сайдбар → «Кастомные поля» → клик по полю в списке
- **Шаги**: те же поля → «Обновить поле» / «Удалить поле»
- **Вариант**: попытаться удалить системное поле (`Стадия`) — должно быть запрещено

## Этап 4 — Telegram подключение (личные чаты)

### US-9 · Подключить личный Telegram для CRM-чатов
- **Открыть**: сайдбар → «Настройки» → «Синк ТГ-папок» (пункт сайдбара «Синк ТГ-папок» раскрывается в этот экран)
  - Альтернативный вход: сайдбар → «Чат», empty-state «Синхронизируйте папки Telegram» / «Автоматическая синхронизация»
- **Шаги**:
  1. «Подключить Telegram»
  2. По умолчанию **QR**: «Откройте Telegram на вашем телефоне → Настройки > Устройства > Подключить устройство → Отсканируйте этот QR-код»
  3. Альтернатива — «Подключить по номеру телефона»: номер → код → (если 2FA) пароль
  4. Успех → появляются папки / чаты
- **Privacy-обещания в UI**: «Выборочная синхронизация», «Зашифрованный токен сессии», «Только метаданные: Мы импортируем только метаданные чата (имя, участники, время последнего сообщения), но не содержимое сообщений»
- **RPC**: `telegram.client.sendCode` · `signIn` · `signInWithPassword` · `status` · `getFolders` · `triggerSync` · `getQrState` · `signOut`
- **Варианты**: QR vs код; с 2FA vs без
- *(URL: `/settings/telegram-sync`)*

### US-10 · Чаты (работа с перепиской) ⚠️ Firestore-heavy
- **Открыть**: сайдбар → «Чат»
- **Базовые состояния**:
  - Empty: «Выберите чат» (когда ни один не открыт)
  - Список чатов слева, контент чата справа
- **Действия для прокликивания (чтобы triggerнулись Firestore-пакеты и capture записал снапшоты)**:
  1. **Открыть чат** — клик по чату в списке слева. Ожидается: `_firestore.html` снапшот с loaded-перепиской
  2. **Переключиться на другой чат** — клик по другому чату. Ожидается: контент справа обновился
  3. **Отправить текстовое сообщение** — набрать в input → Enter / кнопка отправки
  4. **Получить входящее сообщение** — попросить собеседника написать (или написать с другого TG-клиента). Ожидается: новое сообщение появилось в тредe без reload
  5. **Скролл истории вверх** — подгрузка старых сообщений (Firestore pagination)
  6. **Read receipt** — переключение с непрочитанного чата: проверить что непрочитанные счётчики обновляются
  7. **Reply / Forward / Edit / Delete** — если кнопки есть в UI, прокликать; иначе зафиксировать что их нет
  8. **Attach / emoji / voice** — если поддерживается, проверить
  9. **Создать лид из чата** — из inventory `ProtectedIndexRouteImport` видны кнопки: «Создать лида в CRM», «Lead Menu», «Редактировать лид», «Добавить описание», «Добавитьзаметку», «Добавитьнапоминание» (это карточка лида справа когда чат привязан)
  10. **Перейти в карточку контакта** — клик по «Редактировать лид» / имени
  11. **«Открыть в Telegram»** — sidebar-кнопка (deep-link в native TG)
- **RPC (HTTP)**: `telegram.account.getAccountConnectionData` · `proxy.getProxyStatus` · `activity.scheduleCalendarEventIfPossible`
- **Основной канал данных**: Firestore subscriptions (messages, chat metadata, unread counters). Через capture: rate-limit 1/сек триггерит снапшот на каждый Firestore-пакет на этом роуте — после прокликивания смотрим `processed/routes/ProtectedWWorkspaceIdTelegramRouteImport/*_firestore.html` для извлечения реального словаря чата
- *(URL: `/w/{id}/telegram`)*

## Этап 5 — CRM: контакты и активности

### US-11 · Список контактов (List + Pipeline view)
- **Открыть**: сайдбар → «Лиды» → **«Список»** (table) или **«Воронка»** (kanban)
- **Элементы экрана**: «Поиск», «Очистить», «Выбрать» (bulk), «Фильтр», «Настройки представления», «Новый лид», «Сортировка: По умолчанию», «Столбцы», «Скрыть пустые столбцы», «Сбросить»
- **Pipeline view**: kanban по property `Стадия` — колонки **`Лид / Разговор / Предложение / Переговоры / Сделка закрыта`** (те же что в сайдбаре «Воронка»)
- **Empty pipeline state**: «Let's setup a pipeline / Start by creating a property to define your pipeline stages / Create property»
- **Save view**:
  1. «Сохранить представление для всех» → модалка «Вы хотите обновить текущее представление или создать новое?»
  2. Ветка «Обновить представление» — перезаписывает текущее
  3. Ветка «Сохранить как новое представление» — открывает форму «Название представления» / «Отменить» / «Создать представление»
  4. Toast: «Представление сохранено»
- **Empty contacts**: «Создайте свой первый лид / Синхронизируйте папки Telegram / Или добавьте вручную»
- **RPC**: `contact.bulkUpdate` · `contact.deleteContacts` · `workspaces.getMembers` (views — Firestore)
- *(URL: `/w/{id}/contacts`)*

### US-12 · Создать лида (3 способа)
- **Открыть**: сайдбар → «Лиды» → «Список» → «Новый лид»
- **Шаги**: модалка «Выберите, как создать новый лид»:
  1. **«Форвадни сообщение»** — переслать сообщение в TG-бот → вручную подтянуть контакт
  2. **«Сканировать QR»** → US-13
  3. **«Добавить вручную»** → форма
- **Форма (вручную)**: «Имя», «Имя пользователя Telegram» (плейсхолдер `например, durov`), «Краткое описание», «Описание», «Стадия» (dropdown воронки), «URL», «Email», «Телефон» + кастомные поля + «Новое поле»
- *(URL формы: `/w/{id}/contacts/new`)*

### US-13 · Создать контакт из QR
- **Открыть**: Лиды → Список → «Новый лид» → «Сканировать QR»
- **Шаги**: сканировать / вставить QR-данные
- **RPC**: `contact.createContactFromQr`

### US-14 · Карточка контакта — просмотр и редактирование
- **Открыть**: сайдбар → «Лиды» → «Список» → клик по строке
- **Действия на карточке**:
  - «Редактировать», «Open Menu» (Lead Menu: «Удалить»)
  - «Открытьчат», «Добавить описание»
  - «Добавитьзаметку», «Добавитьнапоминание» (US-15)
  - «Создать лид» (да, прямо из карточки контакта — проверить семантику)
- **Экран редактирования**: те же поля что в US-12 форме + timeline активностей
- **RPC**: `contact.updateContactAvatar` · `telegram.account.getAccountConnectionData` · `proxy.getProxyStatus`
- *(URLs: `/contacts/{contactId}`, `/contacts/{contactId}/edit`)*

### US-15 · Активности — заметки и напоминания
- **Открыть**: карточка контакта (US-14) → «Добавитьзаметку» или «Добавитьнапоминание»
- **Заметка**: поле «Заметка...» + кнопки `Format bold`/`Format italic` → «Создать заметку»
- **Напоминание**: «Дата» (`April 22nd, 2026`), «Повторять...», «Описание...» → «Создать напоминание»
- **В timeline**: у каждой активности «Menu» → «Редактировать напоминание» / «Удалить напоминание» / «Обновить напоминание» / отметить выполненным (`completed today at 5:20 AM`)
- **RPC**: `activity.scheduleCalendarEventIfPossible` (срабатывает если у активности есть дата)
- ⚠️ Редактирование открывается **inline-модалкой** поверх карточки контакта — URL `/contacts/{cid}/activities/{aid}/edit` существует как роут, но через меню туда не попасть (только как deep-link).
- *(URLs: `/contacts/{cid}/activities/new`, `/contacts/{cid}/activities/{aid}/edit`)*

### US-16 · Массовое удаление контактов
- **Открыть**: Лиды → Список → «Выбрать» → выделить чекбоксами
- **Шаги**: bulk-actions → «Удалить»
- **RPC**: `contact.deleteContacts` · `contact.bulkUpdate`

## Этап 6 — Outreach: аккаунты, кампании, лиды

> Названия кнопок/экранов — из реальных HTML-снапшотов (см. `tools/capture/ui-inventory.md`).

### US-17 · Добавить Telegram-аккаунт для рассылок
- **Цель**: подключить TG-аккаунт для outreach (≠ US-9, там личный для CRM-чатов)
- **Открыть**: сайдбар → «Рассылки» → «Telegram-аккаунты» → «Добавить Telegram-аккаунт»
- **Шаги**:
  1. «Номер телефона» → выбрать «Страна прокси» (`Авто` / конкретная, напр. `KZ` / `RU`) → «Отправить код»
  2. Ввести код из Telegram
  3. Успех: «Аккаунт успешно добавлен» → «Просмотреть аккаунт»
- **Монетизация на экране**: блок «Нужны дополнительные аккаунты для рассылок? — Купить Telegram-аккаунты напрямую через нас — $25 / аккаунт»
- **RPC**: `telegram.account.auth` · `telegram.account.authState` · `proxy.getCountries` · `proxy.getProxyStatus`
- **Варианты**: прокси Авто vs страна; разные country-коды; с/без 2FA

### US-18 · Настройки TG-аккаунта
- **Открыть**: сайдбар → «Рассылки» → «Telegram-аккаунты» → клик по аккаунту
- **Блоки экрана**: «Статус» (`Active`), «Страна прокси», «Прогрев аккаунта» (`Остановить` / `Запустить`), «Автосоздание лидов» (`Отключено`), «Дневной лимит лидов», «Выйти»
- **Toast при смене лимита**: «Дневеной лимит обновлен · Все кампании будут перепланированы»
- **RPC**: `telegram.account.toggleWarmup` · `triggerWarmupSession` · `updateAccount` · `outreach.rescheduleSequences` (при смене лимита)
- **Re-auth** (если сессия слетела): `reauthenticateWebClient` · `submitReauthPassword`

### US-19 · Переместить аккаунты между воркспейсами
- **Открыть**: «Рассылки» → «Telegram-аккаунты» → «Выбрать» (bulk-режим) → «Переместить»
- **RPC**: `telegram.account.moveAccounts`
- **Статус**: не прокликано — UI bulk-actions уточнить

### US-20 · Создать кампанию из CRM (с фильтрами / split)
- **Prereq**: US-7 (properties), US-11 (контакты), US-17 (TG-аккаунт)
- **Открыть**: сайдбар → «Рассылки» → «Кампании» → «Новая кампания»
- **Шаги**:
  1. Выбрать источник — **«Лиды из CRM»** (подпись «Использовать лидов из CRM»)
  2. Тип: **«Одноразовая»** (создать один раз) / **«Динамическая»** («Все лиды из CRM, которые удовлетворяют фильтрам, будут автоматически добавляться в эту кампанию»)
  3. «Фильтры» → «Добавить фильтр» → `property == value` (например `Стадия: Разговор`)
  4. «Предварительный просмотр» — показывает кол-во лидов, соответствующих фильтру. Warning если 0: «Нет лидов, соответствующих вашим фильтрам. Здесь отображаются только лиды с именем пользователя Telegram»
  5. Создать → попадает в черновик кампании
- **RPC**: `outreach.lists.createCrmList` · `outreach.sequences.create`
- *(URL: `/sequences/new/crm`)*

### US-21 · Создать кампанию из CSV
- **Открыть**: «Рассылки» → «Кампании» → «Новая кампания» → «Загрузить CSV» («Загрузить CSV файл с лидами»)
- **Шаги**:
  1. «Загрузите CSV файл с вашими лидами» → «Выбрать файл»
  2. «Сопоставьте поля со столбцами» — дропдауны «Столбец с username» / «Столбец с телефоном» («Выберите, где в файле находятся username и телефон. Достаточно указать только одно поле.»)
  3. «Предупреждение: Столбцы без заголовка были пропущены»
  4. «Предпросмотр» первых строк → «Продолжить»
- **Ограничения при рассылке по номерам (warning в UI)**: «Проверка дубликатов недоступна по номерам телефонов», «Некоторые получатели могут быть недоступны из-за их настроек приватности»
- **RPC**: `outreach.generateUploadSignedUrl` · `outreach.lists.uploadCsvList` · `outreach.sequences.create`
- *(URL: `/sequences/new/csv`)*

### US-21a · Создать кампанию из «Группы из CRM»
- **Открыть**: «Рассылки» → «Кампании» → «Новая кампания» → «Группы из CRM» («Отправить рассылку по группам»)

### US-22 · Настроить сообщения кампании
- **Открыть**: «Рассылки» → «Кампании» → клик по кампании → таб «Детали кампании» → «Outreach checklist» → «Добавить ваше первое сообщение»
- **Редактор**: текстовое поле + `Format bold` / `Format italic` + «Insert variable» (Typeahead menu с переменными, напр. `​Username​`)
- **Цепочка**: «Добавить сообщение» → «Через N день отправить текст» (задержка)
- **Превью**: «Предпросмотр сообщения» → «Предпросмотр для лида» → «Случайный» (подставляет реального лида, напр. `@mikeshestakov_dev`) → «Закрыть»
- **RPC**: `outreach.validateTextVariables` · `outreach.sequences.patch`
- **Варианты**: с переменными / без; с вложениями (ждём `generateUploadSignedUrl`)

### US-23 · Привязать TG-аккаунты к кампании
- **Открыть**: внутри кампании → таб «Аккаунты»
- **Шаги**: radio «Использовать все аккаунты» / «Использовать выбранные аккаунты» → чекбоксы по списку аккаунтов → «Сохранить»
- **RPC**: `outreach.sequences.patch`

### US-24 · Настройки создания лидов в CRM (contact-settings)
- **Открыть**: внутри кампании → таб «Настройки создания лидов в CRM»
- **Блоки**:
  1. **«Создавать контакт в CRM»** — radio: «Когда лид ответит» / «Когда первое сообщение отправлено»
  2. **«Ответственные по умолчанию»** — поиск по пользователям воркспейса (см. `/contact-settings/owners` — round-robin / по правилу)
  3. **«Значения по умолчанию для новых лидов в CRM»** — «Укажите значения по умолчанию для новых лидов в CRM, созданных из этой кампании». Для `Стадия` — дропдаун воронки + «Настроить список» (открывает inline-редактор опций: «Новое значение» / «Изменить цвет» / «Удалить»)
- **Toast**: «Настройки создания контакта сохранены»
- **RPC**: `outreach.sequences.patch` · `workspaces.getMembers`

### US-25 · Управление лидами кампании
- **Открыть**: внутри кампании → таб с лидами (таблица `Лид / Аккаунт / Telegram / Действия`)
- **Элементы**: тоггл «Показать CSV данные» (разворачивает колонки CSV: `first_name`, `last_name`, `email`, `company`, `position`, `city`, `country`, `industry`, `deal_size`, `lead_source`, `interest_level`, `notes`)
- **Действия**: убрать лид из кампании, отправить сейчас, обновить property, resolve duplicates
- **RPC**: `outreach.getLeads` · `removeLeadFromSequence` · `sendOutreachMessageNow` · `updateLeadProperties` · `resolveDuplicates`

### US-26 · Запуск/пауза/статистика/аналитика/удаление
- **Открыть**: внутри кампании → таб «Детали кампании»
- **Действия**:
  - «Запустить» / «Приостановить» (статусы: `Черновик` → `Активна` → `На паузе`)
  - «Удалить кампанию»
- **Вкладки**:
  - «Детали кампании»
  - «Обработка» (подпись: «Отслеживайте прогресс · Здесь вы можете увидеть все запланированные сообщения, а также какие-либо ошибки»; подсказки: «Перезапуск кампании часто помогает при любой ошибке», «Вернитесь в раздел Рассылки и проверьте Расписание кампаний»)
  - «Статистика» («Отправлено» / «Прочитали» / «Ответили»)
  - «Аналитика» («Период»: `7 дней` / `30 дней` / `90 дней` / `Свой`; группировка `По дням` / `По неделям` / `По месяцам`; «Отображать»: `По дате отправки` / `По дате события`; календарь-пикер с выбором диапазона)
- **RPC**: `outreach.updateSequenceStatus` · `getSequenceStats` · `getSequenceAnalytics` · `rescheduleSequences` · `outreach.sequences.delete`

### US-27 · Глобальное расписание отправок

- **Шаги**: дни недели × часы × таймзона
